Criptografia

Proteção de Dados & Comunicação Segura

// O que é Criptografia

Criptografia é a ciência de transformar informação legível — chamada de texto claro (plaintext) — em um formato ilegível denominado texto cifrado (ciphertext), de modo que apenas entidades autorizadas consigam reverter essa transformação e recuperar os dados originais.

A palavra vem do grego: kryptós (oculto) + graphein (escrever). Usada há milénios — das cifras de César ao Enigma da Segunda Guerra —, hoje é a fundação invisível que protege senhas, transações bancárias, mensagens e toda a infraestrutura crítica da internet moderna.

Dois conceitos-chave sustentam a criptografia contemporânea:

Algoritmo A função matemática que define como cifrar e decifrar. Ex.: AES-256, RSA-2048, ChaCha20.
Chave O segredo que parametriza o algoritmo. Sem a chave correta, mesmo conhecendo o algoritmo, o texto cifrado é computacionalmente inquebrável.

// Simétrica vs Assimétrica

Existem dois paradigmas principais que diferem fundamentalmente na forma como as chaves são geridas e distribuídas entre as partes comunicantes.

Tipo 01

Criptografia Simétrica

Uma única chave secreta é usada tanto para cifrar quanto para decifrar os dados. Ambas as partes precisam partilhar essa chave de forma segura antes de qualquer comunicação.

  • Extremamente rápida — ideal para grandes volumes
  • Problema: distribuição segura da chave
  • Algoritmos: AES, ChaCha20, 3DES
  • Uso: criptografia de disco, VPNs, bases de dados
Tipo 02

Criptografia Assimétrica

Opera com um par de chaves matematicamente ligadas: a chave pública (partilhável com todos) para cifrar, e a chave privada (secreta) para decifrar.

  • Resolve o problema da distribuição de chaves
  • Mais lenta — usada para troca de chaves e assinaturas
  • Algoritmos: RSA, ECC, Ed25519
  • Uso: TLS/HTTPS, SSH, assinaturas digitais, PGP

Na prática, os sistemas modernos utilizam criptografia híbrida: a assimétrica para trocar com segurança uma chave de sessão, e a simétrica para cifrar o tráfego real com essa chave — combinando a segurança de uma com a velocidade da outra.

// Como o HTTPS Protege Dados em Trânsito

HTTP transmite dados em texto claro — qualquer ponto intermediário na rede pode ler ou modificar o conteúdo. HTTPS = HTTP sobre TLS (Transport Layer Security), que envolve a comunicação num canal criptografado e autenticado.

BROWSER ──────────────► SERVIDOR
ClientHello: versão TLS, cifras suportadas, nonce aleatório
BROWSER ◄────────────── SERVIDOR
ServerHello + Certificado X.509 (contém chave pública)
BROWSER ──────────────► SERVIDOR
Segredo pré-mestre cifrado com a chave pública do servidor
BROWSER ◄══════════════► SERVIDOR
✓ Canal AES-256 estabelecido — todos os dados cifrados daqui em diante

O certificado TLS é emitido e assinado por uma Autoridade Certificadora (CA) de confiança, garantindo que o servidor é realmente quem diz ser — prevenindo ataques man-in-the-middle. O cadeado verde no browser é a confirmação visual deste processo.

A versão atual é o TLS 1.3, que elimina algoritmos inseguros legados, reduz a latência do handshake (1-RTT ou 0-RTT) e torna obrigatório o Perfect Forward Secrecy (PFS) — garantindo que, mesmo que a chave privada do servidor seja futuramente comprometida, as sessões passadas permanecem protegidas.
